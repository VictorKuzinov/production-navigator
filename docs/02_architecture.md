# Архитектура проекта

## Общая логика системы

Производственный профиль
        ↓
Анализ возможностей
        ↓
Подбор заказов
        ↓
Рекомендации

## 1. Производственный профиль

Назначение:
хранение информации о производственных возможностях предприятия без сбора лишних персональных данных.

### Продукция

- что производится сейчас;
- серийная продукция или изделия под заказ;
- наличие каталога продукции.

### Оборудование

- типы станков и оборудования;
- наличие ЧПУ;
- рабочая зона;
- максимальная масса детали.

### Материалы

- сталь;
- нержавеющая сталь;
- алюминий;
- латунь;
- пластик;
- другие материалы.

### Технологии

- токарная обработка;
- фрезерование;
- лазерная резка;
- сварка;
- покраска;
- другие технологии.

### Дополнительные параметры

- размеры деталей;
- допуски;
- объемы производства;
- сертификаты;
- производственные площади.

## 2. Анализ возможностей

На основании производственного профиля система анализирует, какую продукцию предприятие может производить и какие ограничения необходимо учитывать.

## 3. Подбор заказов

Система подбирает заказы, которые соответствуют производственному профилю предприятия.

Подбор выполняется по следующим параметрам:

- продукция;
- материалы;
- технологии;
- размеры деталей;
- объемы;
- наличие необходимых документов и сертификатов.

## 4. Рекомендации

Система формирует рекомендации по наиболее перспективным направлениям.

Рекомендации могут включать:

- топ подходящих заказов;
- рекомендуемую продукцию;
- оценку перспективности направления;
- возможные ограничения;
- необходимые действия для участия в закупках;
- рекомендации по развитию производственных возможностей.

В дальнейшем система может работать в режиме регулярного мониторинга 
рынка и по подписке отправлять пользователю подходящие предложения.

## 5. Принципы проекта

- система собирает только данные, необходимые для анализа производственных
- возможностей;
- персональные данные в MVP не собираются;
- сначала описываются возможности производства, затем подбираются
- заказы;
- рекомендации должны быть объяснимыми;
- система должна развиваться поэтапно: от простого профиля к аналитике
- и CRM.
