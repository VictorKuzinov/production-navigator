# Производственный навигатор

## 1. Введение

**Производственный навигатор** — это цифровой консультант для производственного предприятия, который помогает понять возможности производства, найти подходящие заказы и определить направления развития бизнеса.

Девиз проекта:

> Находим не тендеры, а возможности для производства.

## 2. Цель проекта

Цель проекта — создать систему, которая помогает малым и средним производственным предприятиям:

- описать свои производственные возможности;
- систематизировать данные об оборудовании, продукции и технологиях;
- находить подходящие заказы и закупки;
- получать рекомендации по развитию производства;
- выявлять слабые места, мешающие получению заказов.

## 3. Проблема рынка

Многие небольшие производственные предприятия имеют оборудование, опыт и персонал, но при этом испытывают сложности с поиском заказов.

Основные проблемы:

- нет системного анализа производственных возможностей;
- информация об оборудовании и продукции хранится разрозненно;
- предприятие не понимает, какие заказы ему подходят;
- участие в закупках требует подготовки документов и опыта;
- нет понятного плана развития продаж и продвижения;
- простаивают мощности.

## 4. Для кого предназначена система

Система предназначена для:

- малых производственных предприятий;
- мастерских с ЧПУ;
- предприятий металлообработки;
- производителей нестандартных изделий;
- компаний, которые хотят выйти на рынок госзакупок;
- руководителей, которым нужен понятный план развития производства.

## 5. Основные возможности

Первая версия системы должна включать:

- анкету предприятия;
- учет оборудования;
- описание производственных возможностей;
- описание продукции;
- анализ готовности к участию в закупках;
- поиск подходящих заказов;
- формирование рекомендаций;
- подготовку основы для дальнейшей CRM-системы.

## 6. Архитектура проекта

На первом этапе проект планируется реализовать как Python-приложение с простым интерфейсом.

Предварительный стек:

- Python;
- Streamlit;
- SQLite;
- SQLAlchemy;
- pandas;
- requests;
- AI-модуль для анализа и рекомендаций.

Предполагаемая структура проекта:

```text
production-navigator/
├── backend/
├── docs/
├── frontend/
├── prototype/
└── tests/
```

## 7. Этапы развития

Проект планируется реализовывать поэтапно.

### Этап 1. Концепция

- формирование идеи проекта;
- описание целей и задач;
- разработка архитектуры;
- проектирование базы данных.

### Этап 2. MVP

- карточка предприятия;
- анкета предприятия;
- учет оборудования;
- описание продукции;
- производственные возможности;
- рекомендации.

### Этап 3. Поиск заказов

- поиск государственных закупок;
- поиск коммерческих заказов;
- фильтрация по возможностям предприятия;
- сохранение интересующих закупок.

### Этап 4. Аналитика

- анализ загрузки предприятия;
- анализ производственных возможностей;
- рекомендации по развитию;
- оценка конкурентоспособности.

### Этап 5. CRM

- клиенты;
- коммерческие предложения;
- договоры;
- счета;
- история взаимодействия.

### Этап 6. AI-помощник

- анализ предприятия;
- рекомендации по развитию;
- помощь в подготовке заявок;
- ответы на вопросы пользователя.

### Этап 7. Экосистема

- интеграция с внешними сервисами;
- обмен данными;
- мобильное приложение;
- облачная версия.

## 8. MVP

MVP — минимальная рабочая версия системы.
В MVP должны войти:
- карточка предприятия;
- анкета производственных возможностей;
- справочник оборудования;
- справочник продукции;
- первичный анализ предприятия;
- рекомендации по следующим шагам;
- ручной или полуавтоматический поиск подходящих закупок.

## 9. Дальнейшее развитие

В дальнейшем система может быть расширена следующими модулями:
автоматический парсер закупок;
CRM для клиентов и заказов;
- модуль подготовки коммерческих предложений;
- модуль анализа конкурентов;
- модуль работы с документами;
- интеграция с внешними площадками;
- AI-помощник для руководителя производства.

## 10. Итоговое видение

В перспективе Производственный навигатор должен стать не 
просто программой для поиска тендеров, а полноценной системой
поддержки развития производственного предприятия.
Система должна помогать отвечать на ключевые вопросы:
- что предприятие реально может производить;
- какие заказы ему подходят;
- чего не хватает для выхода на новые рынки;
- какие действия нужно выполнить в первую очередь.
